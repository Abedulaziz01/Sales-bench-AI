# Methodology Rationale - Path B Preference-Tuned Judge

Path B is the right intervention because the Week 10 failures are mostly evaluation failures, not raw language-generation failures. In `TRACE-002`, the agent produces fluent outreach but collapses into formulaic phrasing, banned Tenacious language, and prospect-facing use of "bench." In `TRACE-003`, the draft stays readable yet still misqualifies the account and implies unsupported capacity. In `TRACE-005`, the message references a real leadership-change signal but drifts into canned staffing language that breaks the Tenacious tone markers. Those traces all point to the same diagnosis: the baseline can often write acceptable English, but it cannot reliably tell when an output violates Tenacious-specific rules around grounded claims, honest capacity, professional language, and non-condescending framing.

That failure profile matches the preference-learning setup in Direct Preference Optimization (Rafailov et al., 2023): the useful supervision signal is not "write more text like this" but "prefer this grounded, policy-compliant draft over this superficially plausible failure." Our training pairs are built exactly that way. Rejected outputs come from Week 10 trace failures and preserve the observable failure modes the production agent actually exhibited. Chosen outputs are corrected rewrites that satisfy the Tenacious guide: one ask, no banned phrases, no prospect-facing "bench" language, confidence-aware phrasing on weak signals, and refusal to over-commit capacity or fabricated pricing.

For the optimizer choice, the implementation rationale is closer to SimPO/ORPO-style lightweight preference learning than to a large reference-heavy RL stack. SimPO (Meng, Xia, and Chen, 2024) is especially relevant because our setting is small-data, low-cost, and preference-separable: we care about ranking a compliant draft above a non-compliant one, not about recovering a broad reward model over open-ended chat. Prometheus 2 (Kim et al., 2024) also supports the judge-first framing. Its central lesson for this project is that a smaller model can become useful as an evaluator when the training signal is tightly coupled to explicit rubric dimensions. That maps directly onto our five Tenacious tone markers and the mechanical pre-flight checks in the evaluator.

Preference leakage is the main methodological risk for Path B, so the pair-construction pipeline uses model-family separation. Rejected samples come from `week10/trace_log.jsonl`, chosen rewrites are generated under a simulated DeepSeek-family rewriter, and the quality gate is a separate Qwen-family judge. That separation follows the warning in Preference Leakage (Li et al., 2025): if the same family writes and grades the rewrite, the learned preference can encode model-specific style rather than the business rule we actually care about. We reserve any eval-tier spot-checking for Days 5-6 only, which keeps Day 4 preparation aligned with the cost and leakage constraints.

The contamination design also follows the failure mode. Because the training artifact is a preference-pair file rather than plain tasks, the correct test is training-pair text against both `dataset/dev/tasks.jsonl` and `dataset/held_out/tasks.jsonl`. The committed report in `training/train_contamination_check.json` records `0` n-gram violations and `0` embedding violations against held-out after pair construction, so the judge is trained on Tenacious-style failures without leaking the sealed evaluation slice into the preference data.
